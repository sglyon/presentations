Julia

What is Julia and why should I care? What is Julia good at?

  • An exciting new language made specifically for scientific computing
  • Interpreted (like Matlab/Python), but also (JIT-)compiled
  • Easy to learn, write, and maintain, but designed with performance in mind (2-3x C speed)
    • Define functions wherever you want
    • Functions with optional, keyword, defualt arugments
    • Organize code in modules/packages
  • Core concept is multiple dispatch -- specialize function behavior based on types of all arguments (more later)
  • Parallel programming features built into the language - easy and inexpensive scalability (more later)
  • Most of base Julia is written in Julia itself
    • User code is set on "same level" as language and library code
    • With smart algorithms, will have same performance as the language itself
    • If you know Julia, you can look at source and see how the language itself is implemented

Multiple Dispatch

  • Julia has an advanced expressive type system
  • ... but you can only think about it when you want
  • Functions are applied by matching function name and types of arguments
  • The most specific match possible is called

Parallel Computing

  • Building blocks for parallel computaiton are built in to the language
    • Ability to programmatically start additional local or remote processes (addprocs)
    • Operate on distributed arrays (DArray)
    • Shared memory arrays for local processes (SharedArray)
    • Parallel for loops and map
  • Easy and free to install in the cloud (AWS.jl, GCloud.jl)

@parallel for

  • Suppose we wanted to simulate the number of heads in n coinflips
  • We might write the following function:
In [7]: 
function count_heads(n)
    c::Int = 0
    for i=1:n
        c += randbool()
    end
    c
end

count_heads(10)  # warm up compiler
@timeit count_heads(50000000)  # run timing

1 loops, best of 3: 163.64 ms per loop
  • Now consider the equivalent parallel version of this function:
In [8]: 
function count_heads_par(n)
    @parallel (+) for i=1:n
        int(randbool())
    end
end

addprocs(5)  # add five local processes
count_heads_par(10)  # warm up compiler
@timeit count_heads_par(50000000)  # run timing

10 loops, best of 3: 41.80 ms per loop

Things to notice:

  • The parallel version was shorter than the original
  • Ran more than 4 times faster

Case study: solving models in QuantEcon.jl

  • The QuantEcon.jl library has a number of classic Macroeconomic models
  • Most have these common functions defined:
    • bellman_operator: perform one iteration of natural operator defined by the Bellman equation
    • init_values: produce initial values for discretized value function
  • These building blocks suggest a common (if naive) algorithm for solving a model
    • Compute initial guess for value function from init_values
    • Iterate on bellman_operator until you reach a fixed point
  • In code this looks like
In []: 
function solve_vf(m::AbstractModel)
    init = init_values(m)
    f(x) = bellman_operator(m, x)
    return compute_fixed_point(f, init)
end
  • If we happen to know a better routine for solving a specific model, we can implement it as a special case

using PyPlot

function plot_results(results, odu_mods, n=5)
    n_res = size(results, 1)
    inds = int(linspace(1, n_res, n))
    fig, ax = subplots()
    for i in inds
        mod = odu_mods[i]
        bet = round(mod.bet, 2)
        pi_grid = mod.pi_grid
        ax[:plot](pi_grid, results[i], label=L"$\beta=$"*"$bet")
    end
    ax[:set_xlabel](L"Probabililty of 'bad' model ($\pi$)")
    ax[:set_ylabel](L"Reservation wage ($\bar{w}$)")
    ax[:legend]()
    nothing
end
